# Japanese Lessons ŌĆö Claude Code Context

## Project

Docusaurus-based Japanese learning site. Lessons live in `docs/lessons/` under `grammar/`, `vocabulary/`, and `conjugation/`. The `/dictionary` page is powered by `src/data/vocabulary.yaml`, auto-generated by scanning lesson tables. Deployment is automatic to GitHub Pages on push to `main`.

## Dev Commands

| Command | What it does |
|---------|-------------|
| `npm start` | Start dev server at `http://localhost:3003` (runs extract-vocabulary first) |
| `npm run build` | Build static site (runs extract-vocabulary first) |
| `npm run extract-vocabulary` | Sync `src/data/vocabulary.yaml` from lesson tables |
| `npm run extract-vocabulary:force` | Recreate `vocabulary.yaml` from scratch |
| `npm test` | Run unit tests (Vitest) |
| `npm run test:e2e` | Run e2e tests (Playwright ŌĆö starts site automatically) |
| `npm run typecheck` | TypeScript check without emitting |
| `npm run lint:fix` | Lint and auto-fix |

## Critical: Vocabulary Extraction

Run `npm run extract-vocabulary` after **any** change to vocabulary tables in lesson files. The script scans all `.md` and `.mdx` files in `docs/lessons/`, extracts tables with `Hiragana | Kanji | Romaji | English | Type` columns, merges into `src/data/vocabulary.yaml` (preserving IDs and tags), and removes orphaned entries. Use `--force` after major restructuring.

The hook in `.claude/settings.local.json` runs this automatically on lesson file edits.

## File Extension Rules

- `.md` ŌĆö use for all lesson files. Docusaurus processes `.md` as MDX so JSX imports (`NextSteps`) work.
- `.mdx` ŌĆö section index files (`index.mdx`) only, or when importing `LessonList` / `ReferenceCard`.
- When in doubt: `.md`.
